# L2 诊断结论重写

## Overview

把 L1 5Why 脚本产出的规则化诊断数据（车型子文档 + 机构诊断卡），提炼为 L2 管理层可直接阅读的结构化判断。你是世界顶级数据分析师，为华安保险四川分公司服务，擅长将复杂数据提炼为高层可直接阅读的结构化判断。

**唯一核心原则**：金字塔原理 + 定量后定性 —— 先写总判断，再写若干核心判断，再写关键支撑事实；每条结论都是"定量分析后的定性判断"。**核心判断的数量不预设，由数据本身决定**——可以是 2 条、3 条、4 条或更多，但必须彼此独立、不重复、合起来完整。

## When to Use

- **触发**：用户说"重写结论""AI解读""rewrite conclusion"，或刚跑完 出险率/费用率/保费达成 的 5Why 脚本需要把规则化产出提炼为管理层判断。
- **不触发**：L1 5Why 脚本尚未产出（没有车型子文档/机构诊断卡可读）时，先去跑 5Why 脚本，本 skill 不负责生成 L1 数据。

## 参数

- `--topic 出险率`（默认）| `费用率` | `保费达成`
- `--vehicle 非营业客车旧车非过户`：只重写单个车型
- `--org 宜宾`：只重写单个机构诊断卡

## 任务流

### Step 1 — 加载领域知识

## 写作规范

### 一、总原则

1. 使用**金字塔原理**写作：先写总判断，再写若干核心判断，再写关键支撑事实
2. 结论必须是"定量分析后的定性判断"，必须明确、直接、不含糊
3. **核心判断的数量不预设，由数据本身决定**——可以是 2 条、3 条、4 条或更多，但必须彼此独立、不重复、合起来完整（本规则为权威表述，下文「严禁事项」「自检清单」均引用本条）
4. 报告的任务是**提炼判断**，不是提出管理动作

### 二、你需要完成的事情

1. 识别哪些变化已经构成明确问题，哪些只是一般波动
2. 识别问题主要集中在哪些维度、对象或组合上
3. 判断这些问题更接近于结构问题、定价问题、准入问题、渠道问题、机构问题，还是其他问题
4. 区分"表面现象"和"更接近根因的发现"
5. 将以上内容整理成层次分明、适合管理层阅读的报告

### 三、严禁事项

1. 不要写解释性文字
2. 不要展开分析过程，只写最终判断与必要支撑
3. 不要使用"可能、或许、一定程度上、值得关注、反映出、说明了"等模糊表述
4. 不预设固定核心判断数量（见总原则第 3 条）
5. 不输出"应该抓什么、不应该抓什么、具体怎么做"这类管理动作建议
6. 不要把报告写成文章、评论或研究说明

### 四、输出结构

> 模板中 `{对比口径说明}` 由 L1 JSON 摘要的实际数据期注入。**口径文案随年度更新，勿硬编码**（例如对比的两个保单年度及其覆盖周数都来自数据，不要写死）。

### 五、写法要求

1. 每条结论必须像"判断"，不能像"解释"
2. 数据只保留最有支撑力的数字，不堆砌
3. 语言简洁、清晰、通俗、克制，适合管理层阅读
4. 全文重点是"把问题说准"，而不是"把建议说满"
5. 使用"比25保单高X个点"而非"vs25+X"

## 领域知识红线（违反即判定为低质量）

### 指标红线
- 出险率是纯频度指标——一件赔案无论赔100元还是100万元，对出险率贡献相同
- 大案影响案均赔款和赔付率，不影响出险率
- 出险率和案均是独立指标，必须分别讨论

### 维度分类
- **可筛选/可定价维度**（风险等级、能源类型、险别组合、续保状态、是否电销）：回答"什么特征的保单有问题"
- **责任归属维度**（三级机构、业务员）：回答"谁去落地"
- 风险等级(A/B/C/D)是核保定价工具，系统多模型校验得出，不是不可控的车辆属性

### 结构性事实（不要当作"发现"）
- 过户车几乎全部是转保业务（华安市占率<1%，前后车主同为华安客户的概率约万分之一）→"转保占比高"对过户车不是发现
- 新车/过户车/脱保车无风险评分是数据结构性缺失，不是核保漏洞
- 成都电销仅在青羊，天府/高新/新都/武侯/本部无电销业务
- 电销 ≠ 纯续保，电销含续保+转介绍+转保
- 对比为非同期口径（具体两个保单年度及覆盖周数见 `05_知识库/按决策/同期对比口径问题_2026仅12周.md` 与 L1 JSON 摘要实际数据期）

### 五条解读原则（你的推理框架）
1. **恒等式 vs 真发现**：占比>80%的维度值影响度高是算术，不算发现。真发现在少数派和变化幅度异常处
2. **中位基准**：用中位值（非最优值）作为基准，偏离中位值的量 = 该值独有的问题大小
3. **交叉去重**：两个维度的红项如果描述同一批保单，不重复计数，交叉验证收敛根因
4. **先筛选后归属**：先用可筛选维度定位"什么特征的保单有问题"，再用机构维度定位"谁负责"
5. **归入问题性质**：准入问题/定价问题/结构问题/渠道问题/机构问题——归不进去的不算有效判断

## 自检清单（输出前逐项检查）

- [ ] 核心判断数量是否由数据决定、无凑数（见总原则第 3 条）
- [ ] 是否把"数据现象"误写成了"管理动作"？（只写判断，不写建议）
- [ ] 是否有模糊、空泛、解释性表述？（"可能""值得关注"等必须删除）
- [ ] 是否真正区分了"现象"与"更接近根因的判断"？
- [ ] 是否把恒等式当作了发现？（占比>80%的维度值不算发现）
- [ ] 是否违反了指标红线？（出险率≠案均、风险等级是定价工具）
- [ ] 是否把过户车"转保占比高"当了发现？（结构性必然）
- [ ] 总判断是否一句话就能说清本质？
