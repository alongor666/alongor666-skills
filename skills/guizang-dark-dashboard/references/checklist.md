# 交付检查清单（深色仪表盘风）

> 生成后逐项核对。P0 必须全过；P1 强烈建议过。脚本能测的交给 ppt-agent 的 `ppt-postflight.mjs`，测不了的肉眼逐页核。

## P0（必过 · 本风格三条教训沉淀）

- [ ] **L1 控件不堆右下角**：`.navhint` 在**右上**（`top:2.6vh;right:2vw`），绝不与页脚右侧栏目说明（`.foot` 第二个 span）在右下角重叠。
- [ ] **L2 内容页填满**：无大片死白。短内容页（卡片少/文字少）必须用「左文 + 右 SVG」双栏，外层 `.row style="flex:1;align-items:center"` 把高度撑开；不要让 `justify-content:center` 把短内容挤成中间一小坨。
- [ ] **L3a SVG 必带标签 + 箭头**：每个节点有文字标签；有向流程/回环用 `<marker>` 标方向。禁"无标签圆点"装饰图。
- [ ] **L3b 标题不折行/不孤字**：中文大标题 `.h-xl` 控制在**一行**；放不下先**缩短文案**，再考虑降字号，禁止出现一个字单独成行（如"…回路"的"路"孤字）。
- [ ] **emoji 白名单**：仅 🟢🔵🟡🔴 + 🔴（红线）。其余 emoji 一律删。
- [ ] **占位符清零**：无 `[必填]`、无 `NN`/`/NN` 残留页码、`<title>` 已改。
- [ ] **数据真实**：所有数字可追溯到真实产物，无编造；多源拼装登记口径。
- [ ] **脱敏**：无真名/车牌/手机号（除非已确认可公开）。

## P1（强烈建议）

- [ ] **页脚三件套**：每页 `.foot` 有「DECK·页码/总数 + 栏目/角色 + (可选)注脚」，页码 `<b>` 高亮当前页。
- [ ] **节奏交替**：相邻页 `class="slide"` / `slide alt` 交替制造呼吸（深色风靠 `.alt` 背景微变，不是靠 light/dark 切换——ppt-postflight 的"同色调"警告对本风格是误报，可接受）。
- [ ] **英文专名括注**：codex / governance / RLS / CI 等首次出现给一句中文释义。
- [ ] **强调色克制**：电蓝(`--accent`)主、青(`--accent2`)辅、琥珀/红仅用于红线与缺陷数；不要满屏高亮。
- [ ] **大字双约束限高**：超大数字/标题用 `min(Xvw, Yvh)`，只用 vw 会在 16:9 溢出。

## P2（锦上添花）

- [ ] 数据大字报数字用 `font-variant-numeric:tabular-nums`（已在 `.stat .big`）。
- [ ] Before-After / 2x2 用 `.panel`/`.gcell` 等宽对照。
- [ ] 命令速查用 `.cmd` 等宽 chip。

## 演讲体检（沿用 ppt-agent Step 4）

- [ ] 每页都"可讲"：不是只有几个词不收口；完成它承诺的受众状态转移。
- [ ] 高潮页（通常 proof 段）有最具冲击力的真实数字。
