为〈目标用户画像〉重做"〈模块名〉"。这是〈产品类型，如：数据密集型经营分析工具，非消费级 SaaS〉。核心诉求：〈一屏要看懂什么 + 要能定位到谁/哪个维度〉。保留全部数据维度与交互前提下，提升信息层级、可读性与品质。

## 目标受众与核心用例
- 受众：〈谁在用、在什么场景、什么设备〉
- 核心用例：1)〈…〉 2)〈…〉 3)〈…〉

## 当前状态与痛点
- 现状：见所上传 current-page.html（忠实还原当前页）。
- 痛点：〈现在哪里难用/难读/难看，列 2-4 条〉

## 必须保留的功能区块与交互（红线级）
〈逐条列 Phase 0 交互清单：页头/筛选/时间视图/双栏联动/Tab/下钻层级/可排序列/分页…每条标清交互方式〉

## 逐维度设计方向
- 排版：〈字体倾向；要避免的通用字体如 Arial/Inter；字号层级〉
- 配色：〈主色 + 点缀色；语义色映射；承诺用 CSS 变量保持一致；避免怯懦大杂色〉
- 动效：〈克制范围，如仅 hover/过渡；禁弹跳闪烁〉
- 背景/质感：〈底色与层级表达方式〉

## 设计系统约束（产出要能落地，来自本项目）
- 语义色：〈primary/success/warning/danger/neutral 实际色值〉——只在传递语义时上色。
- 字体栈：〈正文/数字/标题〉；数字等宽 tabular-nums。
- 组件预设：〈卡片/按钮/表格的视觉规格〉。
- 数值格式：〈率值小数位与 % 位置；金额单位与千分位；件数；空值符号〉。
- 暗色模式：〈是否需要 + 策略〉。
- 默认排序：〈如主题指标从差到好〉。

## 要避免的默认（黑名单）
- ❌ 通用字体（Arial/Inter 等，除非项目就用它）  ❌ 怯懦/大杂色配色  ❌ 无意义渐变/装饰阴影/注意力消耗型动效  ❌ 为美观牺牲信息密度（数据密集型工具尤忌）

## 红线（不要做）
- ❌ 删任何数据维度或交互  ❌ 用设计系统之外的色彩语言  ❌ 率值多于约定小数位 / 给纯计数列乱涂色  〈+ 项目特定红线〉

## 交付物
- 桌面宽屏（〈如 ≥1440px〉）高保真稿，导出 standalone HTML；如有余力补响应式。
- 标注关键状态（hover / 选中 / 下钻展开 / 联动）的视觉表达。
