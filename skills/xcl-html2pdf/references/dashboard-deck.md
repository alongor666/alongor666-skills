# 经营仪表盘皮肤（诊断报告版）· 组件速查

> `report-skin-dashboard.css` 是 xcl-html2pdf 的第二套皮肤——**现代浅色经营仪表盘风**，与「国家地理风」`report-skin.css` 并列二选一。
> 配 `deck-16x9.css`（16:9 机制）使用；本皮肤自带配色 + 字号 + 组件，**无需再叠 `skin-16x9.css`**。
> 套件：`deck-16x9.css` + `report-skin-dashboard.css` + `deck-16x9.js` + `skeleton-dashboard-16x9.html`（起点）。

## 何时用这套（vs 国家地理风）

| 选 | 经营仪表盘风（本皮肤） | 国家地理风（report-skin.css） |
|---|---|---|
| 受众 | 经营管理层、数据复盘、KPI 盯盘 | 对外发布、品牌叙事、文化感报告 |
| 观感 | BI 数据产品：KPI 卡 + 图表优先、克制专业 | 印刷杂志：朱红/吸积金、纸纹、衬线、印章 |
| 强项 | 排名 / 归因 / 进度 / 行动，图表为主、表格降附录 | 叙事、引文、结构隐喻、视觉冲击 |

> 诊断报告（机构盯盘 / 续保 / 赔付 / 渠道复盘）默认用本皮肤。

## 设计变量（改配色只改这里）

```
--ac 单一强调色（界面主色，默认靛蓝 #2b5ce6）  --ac2 深强调  --ink 正文  --dim 次要文字
--panel 卡片底  --soft 浅块底  --line 描边
状态四级（必须四档可辨）：--ok 达标  --watch 关注  --warn 预警  --bad 危险  --press 压力（归因用）
```

## 🔴 诊断报告铁律：进度口径 ≠ 最终留存

**最容易翻车的一条**。两类率值视觉必须区分，否则管理层把"进度"误读成"最终结果"：

| 口径 | 用色 | 何时 |
|---|---|---|
| **最终留存 / 已成熟**（已到期、已完成窗口） | 四级亮灯 `a-ok` / `a-watch` / `a-warn` / `a-bad` | 续保率/达成率已定型，可判好坏 |
| **进度 / 在途**（未到期、进行中窗口） | 中性强调色 `a-prog`（单色，**不亮灯**） | 数据截止早于完成时点，随时间补齐 |

进度页标题/页脚显式标「进度」；附录进度表续保率列不带 🟢🔵🟡🔴。

## 组件语汇

页结构沿用机制层：`.page`（含 `.cover`）→ `.hd` 页眉 / `.inner` 内容 / `.ft` 页脚（`.pg` 页码）。章节用 `.title`(`.t-no`/`.t-main`/`.t-en`) + `.rule`。

| 组件 | class | 用途 |
|---|---|---|
| 封面 | `.cover` + `.kick`/`.h1`(`.hot`)/`.sub`/`.chips`>`.chip`/`.vbox` | 一句话判断 |
| KPI 卡片网格 | `.kpis`>`.kpi`(`.is-bad`/`.is-warn`/`.is-watch`/`.is-ac`)>`.k-num`/`.k-unit`/`.k-lbl`/`.k-desc` | 核心摘要 4 数 |
| 洞察条 | `.insight` | 把数字串成判断 |
| 横向条形 | `.bars`>`.bar`>`.b-name`/`.b-track`>`.b-fill`(色类)/`.b-val` | 机构排名 |
| 条形色类 | `.a-ok`/`.a-watch`/`.a-warn`/`.a-bad`（亮灯）· `.a-press`（归因压力）· `.a-rest`（其余汇总中性）· `.a-prog`（进度中性） | 见上铁律 |
| 图例 | `.legend`>`i`（色块）+ 文本 | 亮灯页必带，编码可辨 |
| 基准三联 | `.bench`>`.bk`>`.bv`/`.bl`（`.gap` 缺口红） | 合计 / 目标 / 缺口 |
| 结论卡 | `.cards`（2 列，放 4 个即 2×2）>`.card`(`.warn`/`.ok`/`.ac`)>`.c-h`/`.c-p` | 问题 / 行动 |
| 推进图 | `.prog`>`.prow`>`.p-name`/`.p-track`>`.p-fill`(`.cur` 当前/`.lead` 标杆) | 阶段累计、节奏 |
| 紧急框 | `.urg` | 派单 / 红色待办 |
| 附录密表 | `.inner.dense2` 内 `.dt`（`tr.sum` 合计行）+ `.dt-h`(`span` 副标) + `.note` | 全量数据，双表/页 |

亮灯表（最终留存）单元格保留源 `🟢🔵🟡🔴`；进度表不带。`.dense2` 专为双表页收紧行高（`.dt` 默认继承 body 1.7 行高过松会溢出）。

## 起一份诊断报告（流程）

1. 复制 `skeleton-dashboard-16x9.html` + `deck-16x9.css` + `report-skin-dashboard.css` + `deck-16x9.js`（勿改前两个机制/皮肤文件）。
2. 主线 ~10 页讲结论（KPI → 排名 → 归因 → 进度 → 推进 → 行动），**关键表转图表**；原始全表收进 `.dense2` 附录（数据一条不少）。
3. 守铁律：进度≠最终留存（色彩区分）、四级亮灯四档可辨 + 图例、数值零篡改、全中文、图例/轴标签中文。
4. 反复跑 `node driver.mjs <你的>.html <页数>` 到全 PASS（填充 80–100%、零溢出、页数符）。
