填充不足怎么办：补实质内容（小数据表 / 补充说明 / 引言），不要靠拉大间距充数。溢出怎么办：精简文字或拆到下一页。验收以 driver.mjs 实测为准。
为什么固定 A4 盒：浏览器自动流式分页会把图表从中间切断、产生半空白页。固定盒 + overflow:hidden + break-after:page，让每页所见即所得——屏幕一屏、打印一张，完全一致。
机制与内容分离：本骨架（xcl-html2pdf）只负责"做成印刷级 PDF 的版面与翻页"；具体报告的结构、配色、图形，由你自己的 <style> 与内容决定。

二、图形层
需要一张图占据下半页时，用 .inner.flexcol + 给图加 class="grow"，它会自动撑满剩余空间。注意把图的 viewBox 比例设到接近渲染比，避免被纵向拉伸变形。

P4 · 收尾
三、收尾层
报告做完，剩下的是验收与导出——把机制兜底交给 driver，把判断留给眼睛。
起一个新报告的三步：
① 复制本骨架与同目录的 page-deck.css、deck.js。
② 把每个 .page 换成你的内容，按需增删页（每页一个 <section class="page">）。
③ 跑 driver.mjs 验收，反复调到每页 80–100%、零溢出、PDF 页数正确。
文件作用
page-deck.cssA4 版面盒 + 屏幕横向翻页 + 打印一页一张（勿改）
deck.js翻页交互（勿改）
你的 .html内容 + 本报告专属 <style>
driver.mjs验收：填充率 / 溢出 / 真实 PDF 页数
截图自查Chrome --headless --screenshot 看真实渲染
页数核对driver 数真实 PDF 的 Page 对象
本地预览open 你的 .html（macOS），屏幕即横向翻页
退出码：driver.mjs 返回 0=全部达标、1=有页不达标、2=运行错误，可直接接入脚本或 CI。
常见坑：file:// 会被部分自动化工具拦截，验收走本地 http（driver 已内置）；flexcol 页的 SVG 若被纵向拉伸变形，把 viewBox 高度调到接近渲染比即可消除。
屏幕上：← → / 点击左右 / 滚轮 翻页。导出：Cmd/Ctrl+P → 另存为 PDF → 取消「页眉和页脚」 → 即得一页一张。
